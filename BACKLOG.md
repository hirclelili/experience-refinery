# BACKLOG — 经历炼金术

> 记录 v1.0 范围外的想法，供未来版本参考。
> 有想法就加进来，不要塞进当前版本。

---

## v1.1 候选

- [ ] **英文简历版**：在生成 resume.txt 时提供切换为英文版的选项，适合外企/跨国公司投递
- [ ] **非互联网行业适配**：目前采访问题和词库偏向互联网，可以增加咨询/金融/快消等方向的参考文件
- [ ] **多段经历模式**：支持用户一次性整理多段经历，最后生成一个汇总的经历库
- [ ] **简历整体诊断**：对用户现有简历做整体点评（格式、内容、亮点分布）

---

## v2.0 JD 解码器（已规划）

- [ ] 读取岗位 JD，与用户已有的 `experiences/*/index.md` 做匹配
- [ ] 生成"JD-经历匹配矩阵"：哪些经历对应 JD 的哪些要求
- [ ] 生成"缺口分析"：JD 要求了但现有经历没有覆盖的能力点
- [ ] 生成针对该 JD 的专业知识补充建议

## v3.0 模拟面试官（已规划）

## v4.0 求职策略师（已规划）

- [ ] 根据用户学校/专业/经历给出岗位推荐
- [ ] 生成暑期实习/秋招的 timeline 建议

## 其他想法（未评估）

- [ ] **语音输入支持**：用户可以直接语音描述经历，不需要打字
- [ ] **自动搜索行业信息**：采访时如果发现用户缺少行业数据，自动搜索补充（如"同行业 DAU 均值"）
- [ ] **微信小程序版**：把交互搬到微信生态，降低使用门槛
