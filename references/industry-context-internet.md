## 一、量化指标词库

### 用户规模类
| 指标 | 全称 | 使用场景 |
|------|------|---------|
| DAU | 日活跃用户数 | 内容/社交/工具类产品的日常运营 |
| MAU | 月活跃用户数 | 整体产品规模衡量 |
| WAU | 周活跃用户数 | 低频产品（如电商、出行） |
| UV | 独立访客数 | 页面/活动流量衡量 |
| PV | 页面浏览量 | 内容消费量、广告曝光 |
| 注册用户数 | — | 增长类活动效果 |
| 付费用户数 | — | 商业化效果 |

### 增长与留存类
| 指标 | 说明 | 注意 |
|------|------|------|
| 次日留存 / D1 | 次日回访用户/新增用户 | 新用户激活质量 |
| 7 日留存 / D7 | 七日回访率 | 产品黏性 |
| 30 日留存 / D30 | 月留存率 | 长期价值 |
| 流失率 | 1 - 留存率 | 有时更直观 |
| 召回率 | 流失后被拉回来的比例 | 运营效果 |
| 复购率 | 电商/订阅类产品 | — |
| 续费率 / 续订率 | 订阅产品 | — |

### 转化类
| 指标 | 说明 |
|------|------|
| 转化率 | 从某行为到目标行为的比例（加购率/付费转化率/注册转化率） |
| CTR | 点击通过率（Click-through Rate） |
| CVR | 转化率（Conversion Rate） |
| 漏斗转化率 | 各步骤之间的转化比例 |
| 完成率 | 任务/课程/活动的完成比例 |

### 商业化类
| 指标 | 说明 |
|------|------|
| GMV | 成交总额（Gross Merchandise Volume） |
| ARPU | 每用户平均收入 |
| LTV | 用户生命周期价值 |
| ROI | 投入产出比（活动/渠道效果） |
| CPA | 每次获客成本 |
| CPM | 每千次曝光成本 |

### 内容/社区类
| 指标 | 说明 |
|------|------|
| 互动率 | 点赞+评论+转发 / 曝光 |
| 完播率 | 视频完整观看比例 |
| 发布率 | 用户主动生产内容的比例 |
| 月活创作者数 | 内容平台核心指标 |
| 内容消费时长 | 人均每日/每周使用时长 |

### 效率类
| 指标 | 说明 |
|------|------|
| 响应时长 | 需求响应/客服响应时间 |
| 研发周期 | 从需求到上线的时长 |
| 错误率 | 流程/代码/操作出错比例 |
| 复用率 | 某方法/工具被其他团队使用的比例 |

## 二、推荐动词库

> 按使用场景分类，选择最能体现你角色的动词。

### 创建/建立类（适合从 0 到 1 的经历）
- 主导、搭建、创立、建立、设计、开发、构建、打造、孵化
- 规划、制定、建设、落地、实现

### 推动/协调类（适合跨部门、项目管理）
- 推动、驱动、协调、对齐、串联、整合、统筹、牵头
- 引导、带动、促成、推进

### 优化/改进类（适合迭代类经历）
- 优化、迭代、重构、精简、改进、提升、升级、打磨
- 修复、调整、重设计

### 分析/研究类（适合数据/用户研究）
- 拆解、分析、挖掘、识别、发现、验证、量化、测试
- 追踪、监测、对比、建模

### 产出/输出类（适合有明确交付物）
- 输出、沉淀、提炼、撰写、制定、归纳、梳理、整理

### 管理/运营类（适合有管理职责）
- 管理、维护、运营、监管、负责、统筹

## 三、禁用表达清单

> 以下表达在简历和面试中要避免，因为它们空洞、没有信息量，或者暗示了被动角色。

### 严重降分的词
| 禁用词 | 原因 | 替代方向 |
|--------|------|---------|
| 深度参与 | "参与"已经很弱，"深度"更是无法量化的形容词 | 说清楚你具体做了什么 |
| 有效推动 | "有效"是自我评价，没有说服力 | 用数据证明有效 |
| 积极配合 | 暗示被动角色 | 说明你主动做了什么 |
| 协助 XX 完成 | 你是助手，不是主角 | 找到你自己的贡献点 |
| 做了一些 | 模糊，没有信息 | 具体说做了什么 |
| 参与了 | 太泛，没有区分度 | 说你的具体职责 |
| 负责参与 | 自相矛盾（负责=有责任，参与=可能没责任） | 选一个 |
| 相关工作 | 什么都没说 | 具体化 |
| 取得了一定成绩 | "一定"是没有结果的逃避 | 说具体数字或事实 |
| 努力 / 认真 | 个人评价，没有说服力 | 用结果说话 |
| 优秀 / 出色 | 自我评价 | 让老板或数据来评 |

### 容易误用的术语
| 错误用法 | 正确理解 |
|---------|---------|
| PRD = 产品文档 | PRD = Product Requirements Document，是特指文档，不要说"写了个 PRD 文档" |
| KPI = 目标 | KPI 是 Key Performance Indicator，要说"完成 KPI 的 XX%" |
| 复盘 | 不是"开了个总结会"，是系统性地回顾——发生了什么/原因是什么/下次怎么做 |
| 增长黑客 | 不是"做了增长"，是用低成本手段实现快速增长，需要有具体策略 |
| 用户画像 | 不是"我知道用户是谁"，是有数据支撑的用户属性/行为/需求的系统化描述 |

## 四、行业黑话对照表

> 帮助用户用对词，在简历和面试中展示行业感。

| 黑话/术语 | 白话解释 | 使用示例 |
|---------|---------|---------|
| 从 0 到 1 | 从无到有 | "从 0 到 1 设计 XX 功能" |
| 冷启动 | 产品/功能刚上线时用户少、数据少的阶段 | "解决创作者社区冷启动问题" |
| MVP | 最小可行版本（Minimum Viable Product） | "设计 MVP 方案，2 周内验证核心假设" |
| PMF | 产品市场匹配（Product Market Fit） | "通过调研验证 PMF，确定方向" |
| 北极星指标 | 团队最重要的那一个指标 | "以创作者 MAU 为北极星指标" |
| OKR | 目标与关键结果（Objectives & Key Results） | "本季度 OKR 已完成 120%" |
| 漏斗 | 用户从接触到转化的各个环节 | "拆解注册漏斗，优化核心卡点" |
| A/B 测试 | 对照实验，对比两个版本的效果 | "设计 A/B 实验验证新版文案效果" |
| 灰度 / 灰度发布 | 先对部分用户开放，逐步扩大 | "功能先灰度 10% 用户" |
| 全量 | 对所有用户开放 | "验证通过后推全量" |
| 沉淀 | 把经验/方法固化成可复用的东西 | "沉淀为团队标准化方法论" |
| 抽象 | 提炼共性，做成通用方案 | "将 3 个业务需求抽象为统一中台能力" |
| 打通 | 让两个原来不连接的系统/数据/流程连接 | "打通用户行为数据与 CRM 系统" |
| 对齐 | 让相关方达成共识 | "与业务方对齐需求优先级" |
| 卡点 | 阻碍的地方 | "识别用户旅程的核心卡点" |
| 链路 | 一个完整的流程/路径 | "优化从曝光到付费的完整链路" |
